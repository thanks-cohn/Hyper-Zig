# ZIGN01D Grading Rubric

## Point categories

- **Build reproducibility (10):** Exact build command, environment notes, and successful local build.
- **Smoke-test proof (15):** Correct smoke commands, PASS output, and no weakened checks.
- **Command behavior (10):** Shell output matches documentation and does not imply missing functionality.
- **Documentation quality (15):** Clear user-facing docs, expected output, and file references.
- **Limitation honesty (10):** Missing features are explicitly named.
- **Code readability (10):** Small, readable, idiomatic changes in the existing style.
- **Minimality (10):** Scope is limited to the assignment or milestone.
- **Source understanding (10):** Student can explain the files and functions changed.
- **Student explanation (5):** Report is written in the student's own words and answers questions directly.
- **AI-assistance disclosure if relevant (5):** Disclosure follows course policy and all generated material is reviewed.

## Automatic failure conditions

An assignment or milestone may fail automatically for any of these:

- broken build;
- removed smoke test to hide failure;
- fake success claim;
- undocumented command;
- committed generated transcripts by accident;
- claimed real internet/SMS/modem support without proof;
- vague “works” statement with no command/output proof.

## Instructor notes

Reward small, proven work. Penalize broad claims that cannot be traced to commands and output. A correct limitation note is evidence of engineering maturity, not a weakness.
