# ZIGN01D 12-Week Course Map

This map treats ZIGN01D as course material. It does not claim future features exist; planned work is marked as planned.

| Week | Topic | Learning goal | Files/docs to read | Command to run | Expected proof/output | Assignment idea | Limitation to discuss |
| --- | --- | --- | --- | --- | --- | --- | --- |
| 1 | Toolchain, QEMU, and first boot | Build the kernel and see QEMU serial output. | `README.md`, `scripts/build.sh`, `smoke/smoke-v0.sh`, `docs/boot-process.md` | `./scripts/build.sh && ./smoke/smoke-v0.sh` | `PASS ZIGN01D V0 smoke` | Write a one-page build proof report. | QEMU proof is not real hardware proof. |
| 2 | UART and first output | Understand polling UART output. | `kernel/console/uart.zig`, `kernel/main.zig`, `docs/v0-status.md` | `./smoke/smoke-v0.sh` | UART boot and shell markers appear. | Trace one printed boot line to source. | UART polling is simple but not interrupt-driven. |
| 3 | Boot breadcrumbs and logs | Read structured boot markers. | `kernel/diag/breadcrumb.zig`, `kernel/log.zig`, `docs/LOGGING_AND_BREADCRUMBS.md` | `./smoke/smoke-v1.sh` | Breadcrumb/log markers appear in transcript. | Identify five markers and explain their subsystem codes. | Breadcrumbs are not a persistent log buffer. |
| 4 | Shell command dispatch | Trace command parsing and output. | `kernel/console/shell.zig`, `docs/COMMAND_REFERENCE.md` | `./smoke/smoke-v1.sh` | Shell commands return expected diagnostic lines. | Add or document a harmless diagnostic command. | Shell parsing is deliberately small. |
| 5 | Machine status and architecture boundaries | Distinguish safe supervisor reads from unsafe machine assumptions. | `kernel/arch/riscv64/cpu.zig`, `docs/V2_MACHINE_BOUNDARY_AUDIT.md` | `./smoke/smoke-v2.sh` | `PASS ZIGN01D V2 smoke` and machine status lines. | Explain why some CSRs are not read. | Machine-mode details are not fully exposed to the supervisor kernel. |
| 6 | Traps, panics, and failure boundaries | Understand trap-vector scaffold and controlled reports. | `kernel/arch/riscv64/trap.zig`, `kernel/panic/panic.zig`, `docs/TRAPS_AND_PANIC.md`, `docs/V3_TIMER_AND_TRAP_AUDIT.md` | `./smoke/smoke-v3.sh` | Trap-test and panic-test smoke-safe output. | Explain why live fault injection is deferred. | Recovery from arbitrary traps is limited. |
| 7 | Timer reads and monotonic checks | Use `rdtime` polling diagnostics. | `kernel/interrupt/timer.zig`, `kernel/console/shell.zig` | `./smoke/smoke-v3.sh` | Time, ticks, and heartbeat output show `rdtime-polling`. | Write a short note on polling versus timer interrupts. | Timer interrupts and scheduler preemption are not enabled. |
| 8 | MMIO and device boundaries | Study guarded MMIO probe policy. | `kernel/device/mmio_probe.zig`, `kernel/device/device.zig`, `docs/V4_GUARDED_MMIO_AUDIT.md` | `./smoke/smoke-v4.sh` | `PASS ZIGN01D V4 smoke` and deferred MMIO lines. | Explain the allowlist and why live reads are disabled. | Device drivers and safe absent-MMIO recovery are not implemented. |
| 9 | Communication scaffolds and honest not-implemented states | Read communication command output without confusing it for networking. | `kernel/comm/comm.zig`, `kernel/comm/net.zig`, `kernel/comm/sms.zig`, `kernel/comm/modem.zig`, `docs/COMM_V0_AUDIT.md` | `./smoke/smoke-comm-v0.sh` | `PASS ZIGN01D COMM V0 smoke` and not-implemented output. | Write a limitation note for `net get` or `sms send`. | No internet, SMS, modem, calls, or Wi-Fi calling exists. |
| 10 | Smoke tests as proof | Learn the proof contract. | `smoke/README.md`, `docs/PROOF_CONTRACT.md`, smoke scripts | `./smoke/smoke-all.sh` | `PASS ZIGN01D full smoke ladder` if the local environment supports all smoke tests. | Add a docs-only smoke check or explain one existing check. | PASS only proves checked output for that run. |
| 11 | Comparative kernel design | Separate current ZIGN01D from future sibling kernels. | `docs/COMPARATIVE_KERNEL_VISION.md` | `./smoke/smoke-docs.sh` | `PASS ZIGN01D docs smoke` | Design a comparison table for a future sibling repo. | Sibling kernels are planned unless a repo exists and has proof. |
| 12 | Student mini-milestone | Make one minimal, documented, smoke-tested change. | `docs/LAB_MANUAL.md`, `docs/ASSIGNMENTS.md`, `docs/DOCUMENTATION_CONTRACT.md` | Build plus the relevant smoke script. | Build passes; smoke proof matches the claim. | Submit final mini-milestone with proof report. | A small proven milestone is preferred over a broad unproven feature. |
